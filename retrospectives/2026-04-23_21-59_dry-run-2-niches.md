# Ретро: Dry Run AUTOPILOT на 2 нишах (итерация 3)

**Дата:** 2026-04-23 21:59
**Длительность:** ~40 мин
**План:** внутренний

## 1. Задача

Мысленно пройти AUTOPILOT как Клод на двух разных нишах: **стоматологическая клиника «Улыбка» (commercial, B2C services)** и **внутренний SalesTrack-дашборд для отдела продаж (non-commercial, Next.js)**. Найти места где Клод застрянет, где плейсхолдеры не подходят, где логика non-commercial обрывается.

## 2. Как решал

Создал две копии шаблона в `/tmp/dry-run-*` через `cp -r`. Прошёл мысленно все 10 шагов AUTOPILOT, отвечая за пользователя реалистично для ниши. Фиксировал места где:
- вопрос задан, но ответ нигде не используется
- плейсхолдер заточен под не-мою нишу
- правило non-commercial описано частично, остальное «повисает»
- шаг 9a не имеет входных данных

После анализа удалил `/tmp/dry-run-*`.

## 3. Решил?

- [x] Да, полностью - dry-run выявил 3 P1 проблемы non-commercial ветки и 2 P2 ограничения плейсхолдеров. P1 исправлены прямо в ходе итерации, P2 вынесены в known limitations.

## 4. Что можно было лучше

Dry-run мысленный, не реальный - это ограничение. Реальный прогон с пользователем из ниши покажет UX-проблемы (темп, усталость, непонимание терминологии), которые я не заметил. Но для v1.0.0 мысленный прогон - достаточная проверка логической связности.

## 5. Как было / как стало

- **Было:** AUTOPILOT говорит «для non-commercial пропусти economics/unit-economics» - что делать с файлом Клод решает сам, пустой плейсхолдер остаётся в проекте.
- **Стало:** явно прописано «поставь заглушку "не применимо для некоммерческого проекта"» в пропускаемых файлах. `INDEX.md` получит пометку. Структура `.business/` остаётся целой, но non-commercial-пользователь видит только применимое.

## Найденные проблемы и правки

### P1 - Non-commercial ветка оставляет файлы в подвешенном состоянии

**DR1. economics для non-commercial**: AUTOPILOT пишет «только costs.md», но unit-economics, revenue, forecast уже существуют как плейсхолдеры. Клод либо пропускает и оставляет заготовки, либо придумывает что с ними делать.

**DR2. marketing для non-commercial**: AUTOPILOT пишет «только channels.md», но funnel, competitors, content не адресованы. content.md для внутреннего tool = обучающие материалы - это применимо, но AUTOPILOT не указывает.

**DR3. assets для non-commercial**: testimonials.md не применим для внутреннего tool / open-source без отзывов, но AUTOPILOT не упоминает.

**Правка:** обновил [AUTOPILOT.md шаг 8 правило 7](../AUTOPILOT.md) - явно описал что делать с каждым файлом в non-commercial режиме: заглушка «не применимо для некоммерческого проекта» с пояснением причины, плюс пометка в `INDEX.md`.

### P2 - Плейсхолдеры заточены под конкретные ниши (в v1.1.0)

**DR4. `marketing/funnel.md`**: таблица «Показы / Клики / Лиды / Квалифицированные лиды / Продажи / Повторные покупки» - классическая e-commerce / SaaS-воронка. Для стоматологии, агентства, консалтинга - воронка другая (карта/рекомендация → звонок → приём → повторные). Плейсхолдер будет заполнен криво или с пометкой «не моя воронка».

**DR5. `products/pricing.md`**: формат «Тариф / Что входит / Цена / Для кого» подходит для SaaS и комплексных продуктов, но не для прайс-листа услуг (стоматология, салон, клиника). Пользователь адаптирует, но плейсхолдер не помогает.

**Оставляем на v1.1.0:** добавить альтернативные плейсхолдеры `funnel.md` и `pricing.md` с ветвлением по типу бизнеса в AUTOPILOT. Требует рефакторинга плейсхолдеров - не блокер для v1.0.0.

### P3 - обзорное наблюдение (не правим)

**DR6. AUTOPILOT 8.N возобновление в середине блока**: если пользователь закрыл после 2-го вопроса блока 4 (из 4 вопросов), `last_completed_step` остался `8.3`. Возобновление скажет «продолжим с блока 4» - но блок 4 уже начат. AUTOPILOT страхует через правило «перечитай файлы `.business/` которые уже существуют» - Клод прочитает частично заполненный annual.md и продолжит с недостающих вопросов. Работает, хотя неявно.
