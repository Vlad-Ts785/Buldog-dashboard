# Установка скилов с аудитом безопасности

**Когда брать:** после онбординга, когда готов расширить возможности Клода.
**Что делает:** ставит базовые рекомендованные скилы + учит тебя проверять любой скил на безопасность.
**Результат:** `.claude/skills/` с нужными скилами.

## Что такое скил

**Скил** - это папка с инструкцией, которую Клод подтягивает сам, когда видит подходящий триггер. Один скил умеет делать дизайн, другой - парсить PDF, третий - проводить workflow от ресёрча до деплоя.

Два уровня установки:
- **Глобальные:** `~/.claude/skills/` - доступны во всех проектах.
- **Локальные:** `.claude/skills/` в корне проекта - только для этого проекта.

🎯 **Правило:** ставь локально, если скил нужен конкретному проекту. Глобально - только то, чем реально пользуешься везде. Чем меньше скилов - тем меньше токенов на триггеры.

## ⚠️ Безопасность

Скил - это **исполняемая инструкция** от автора. Плохой скил может:
- прочитать твой `.env` с API-ключами
- отправить их на внешний URL через `curl`/`wget`
- удалить файлы
- запустить вредоносный бинарник

**Ручной чек-лист перед установкой любого скила:**

1. Открой страницу скила на Smithery или GitHub.
2. Прочитай `README.md` - что делает, зачем, что требует.
3. Открой главный файл (`SKILL.md` или аналог) и пробеги глазами.
4. Поищи опасные паттерны: `curl`, `wget`, доступ к `.env`, отправку данных на внешние URL, `rm -rf`.
5. Посмотри звёзды и автора. Мало звёзд + неизвестный автор = не ставь.
6. Сомневаешься - прогоняй через промпт-аудит ниже.

## Промпт-аудит любого скила

Используй для проверки ЛЮБОГО скила, MCP или пакета из интернета:

```prompt
Я хочу установить скил по ссылке [ВСТАВЬ URL РЕПОЗИТОРИЯ]. Скачай его исходники, прочитай содержимое (README.md, SKILL.md, все .md и .sh файлы) и проверь, нет ли там опасных операций:

- чтения .env или других файлов с секретами
- отправки данных на внешние URL (curl, wget, fetch)
- удаления файлов (rm, unlink)
- запуска внешних бинарников
- подмены существующих файлов проекта

Дай вердикт: безопасно / подозрительно / опасно - и объясни почему по пунктам. Если опасно - НЕ ставь.
```

## Рекомендованные скилы

### 1. Skill Creator от Anthropic

**Что делает:** помогает тебе создавать свои собственные скилы. Задаёшь задачу - Skill Creator спрашивает структуру, создаёт папку и `SKILL.md`, регистрирует триггер.

```prompt
Установи skill-creator от Anthropic в мой текущий проект (.claude/skills/, не глобально).

1. Найди официальный репозиторий Anthropic со skill-creator (на github.com/anthropics).
2. Проведи аудит безопасности (чтение .env, curl/wget наружу, rm).
3. Если безопасно - установи ЛОКАЛЬНО в .claude/skills/skill-creator/.
4. Покажи, как создать свой первый скил: какой триггер, какая команда.
```

### 2. Frontend Design от Anthropic

**Что делает:** учит Клода нормальной типографике, сеткам, отступам и современной эстетике. Без него - типичный bootstrap-лук. С ним - сайт, за который не стыдно.

```prompt
Установи skill frontend-design от Anthropic в мой текущий проект (.claude/skills/, не глобально).

1. Найди официальный репозиторий Anthropic с этим скилом.
2. Аудит безопасности.
3. Если безопасно - поставь локально.
4. Покажи, как его триггерить: по каким ключевым словам Клод сам поймёт что пора.
```

### 3. PDF skill от Anthropic (опционально)

**Когда нужен:** если в бизнесе есть документы (договоры, прайсы, чеки, отчёты).

```prompt
Установи skill pdf от Anthropic в мой текущий проект (.claude/skills/, не глобально). Аудит безопасности → установка → покажи как пользоваться.
```

### 4. Bulletproof (опционально, авторский скил методолога)

**Что это:** 12-этапный workflow от ресёрча до деплоя. Превращает любую задачу в управляемый конвейер: исследование → план → критика → реализация → тесты → ревью → деплой.

**Это опциональный скил автора методологии смысло-кодинга.** Можно пропустить - шаблон работает и без него. Но если хочешь попробовать:

Ссылка на репозиторий: [github.com/artemiimillier/bulletproof](https://github.com/artemiimillier/bulletproof)

```prompt
Сначала проведи аудит скила по ссылке https://github.com/artemiimillier/bulletproof - см. промпт-аудит выше.

Если вердикт «безопасно» - установи ЛОКАЛЬНО в .claude/skills/bulletproof/ (не в ~/.claude/skills/, не глобально).

После установки покажи структуру .claude/skills/bulletproof/ и объясни как пользоваться: какой триггер, какие команды, как запустить 12-этапный workflow.
```

### 5. Один скил со Smithery под твою нишу

Зайди на [smithery.ai](https://smithery.ai) и выбери **один** скил под свою задачу:

- **Лендинг / UI** → Frontend Design / UI Generation
- **Работа с клиентами** → CRM / Email / Slack
- **Парсинг сайтов** → Browser / Scraping
- **Аналитика** → Google Sheets / Notion / Airtable
- **Контент** → Content / Marketing

**Правила:** не ставь всё подряд. Каждый лишний скил жрёт токены. Один-два реально нужных лучше, чем десять «на всякий случай».

Используй промпт-аудит выше, затем установи локально в проект.

## Финальная проверка

```prompt
Покажи мне дерево папки .claude/skills/ в текущем проекте и список всех скилов, которые сейчас доступны Клоду.

Подтверди что ничего из этих скилов не попало в мою глобальную ~/.claude/.

Для каждого установленного скила коротко (1 строка) напиши, по какому триггеру он активируется.
```
