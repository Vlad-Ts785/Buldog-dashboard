## Найденные проблемы

### P0 - критично, правим первыми

**P0-1. README врёт про защиту `.business/` в `.gitignore`.**

[README.md:83](../README.md#L83): «`.env` и `.business/` автоматически в `.gitignore`». Фактически `.business/` **НЕ** в `.gitignore` изначально (см. [.gitignore:8-10](../.gitignore#L8-L10) - это сделано намеренно для плейсхолдеров). Пользователь, прочитавший README, думает что его бизнес-контекст защищён - но это не так до выполнения шага 10a AUTOPILOT. Риск: пользователь может начать заполнять `.business/` и делать промежуточные коммиты **до** AUTOPILOT-10a, и бизнес-данные утекут в публичный репо.

**Правка:** переформулировать «Клод добавит `.business/` в `.gitignore` автоматически на последнем шаге онбординга - до этого не делай `git commit`».

**P0-2. AUTOPILOT шаг 10a не снимает `.business/` с git tracking.**

Все 29 файлов `.business/` tracked в git (проверено `git ls-files`). Пользователь, склонировавший шаблон через «Use this template», получает `.business/` с плейсхолдерами как tracked. AUTOPILOT 10a добавляет `.business/` в `.gitignore`, но **не делает** `git rm -r --cached .business/`. Итог: после первого `git add -A` на 10a заполненные данными файлы попадут в коммит.

**Правка:** в [AUTOPILOT.md:365-374](../AUTOPILOT.md#L365-L374) добавить шаг: `git rm -r --cached .business/` **перед** `git add -A`.

**P0-3. `retrospectives/README.md` ссылается на несуществующий файл.**

[retrospectives/README.md:25](../retrospectives/README.md#L25): «См. `HELLO-test.md` - первое ретро, созданное автоматически в конце онбординга. Это твой образец.» Но AUTOPILOT 9b и prompts/setup/07-test-cycle.md создают файл с именем `YYYY-MM-DD_HH-MM_hello-test.md`. Пользователь, только что склонировавший шаблон, откроет retrospectives/README.md и не найдёт пример.

**Правка:** переписать абзац на «См. `TEMPLATE.md` как заготовку. Первый реальный пример появится после шага 9b AUTOPILOT под именем `YYYY-MM-DD_HH-MM_hello-test.md`».

### P1 - серьёзно, правим в ту же итерацию

**P1-1. Дубляж `monthly.md` в `goals/` и `execution/`.**

- [.business/goals/monthly.md](../.business/goals/monthly.md) - фокус месяца (1-3 пункта).
- [.business/execution/monthly.md](../.business/execution/monthly.md) - почти идентичный, плюс «как связано с кварталом» и «итог месяца».

[CLAUDE.md:99](../CLAUDE.md#L99) и [prompts/methodology/weekly-planning.md:16](../prompts/methodology/weekly-planning.md#L16) читают `goals/monthly.md`. [.business/execution/README.md:18](../.business/execution/README.md#L18) ссылается на `monthly.md` (относительно = execution/monthly.md). Клод не знает какой из них «источник истины».

**Правка:** удалить `execution/monthly.md`, привести `goals/monthly.md` к более операционному формату (добавить «итог месяца» туда), поправить execution/README на ссылку `.business/goals/monthly.md`.

**P1-2. README обещает «20+ промптов», фактически 16.**

[README.md:15](../README.md#L15). [AUTOPILOT.md:391](../AUTOPILOT.md#L391) и [CHANGELOG.md:23](../CHANGELOG.md#L23) говорят корректно «16».

**Правка:** README - «16 готовых промптов».

**P1-3. Ссылка на личный репо автора методологии с искажённым написанием.**

[prompts/setup/08-skills-install.md:92,95](../prompts/setup/08-skills-install.md#L92): `github.com/artemiimillier/bulletproof`. Написание `artemiimillier` (c лишней `i`) ≠ реальный handle автора. Плюс URL вообще может быть битым - не проверялся. Это привязка к автору методологии, в open-source так делать нельзя - либо убрать, либо заменить плейсхолдером.

**Правка:** заменить на нейтральное «если у тебя есть ссылка на репо Bulletproof от автора методологии, проведи аудит и установи - иначе пропусти». Конкретный URL убрать.

**P1-4. AUTOPILOT шаг 1: вопрос «Ты из России?» нигде не используется.**

[AUTOPILOT.md:37](../AUTOPILOT.md#L37): «Ты из России? Это важно для VPN-проверок и альтернатив.» Но ответ нигде не применяется в шагах 2-10. Пользователь ответил - и всё, ветвления нет.

**Правка:** либо убрать вопрос, либо добавить «если `country: ru` - напомни про VPN перед шагом 4». Предпочтительно - убрать, TROUBLESHOOTING уже покрывает VPN.

**P1-5. Противоречие в обработке `non-commercial` в плейсхолдерах `.business/`.**

[.business/economics/unit-economics.md](../.business/economics/unit-economics.md) пишет в подсказках: «Если `non-commercial`: переименуй в `costs-only.md`». Но AUTOPILOT шаг 8 блок 5 говорит: «Для `non-commercial` - только costs.md, пропусти unit-economics, revenue, forecast». То есть файл `unit-economics.md` не должен заполняться вообще - зачем его переименовывать. Аналогично в [.business/products/pricing.md](../.business/products/pricing.md): «Для `non-commercial`: переименуй в `distribution.md`». AUTOPILOT говорит или `pricing.md` или `distribution.md`. Конфликт: два файла или один переименованный? Клод запутается.

**Правка:** убрать из плейсхолдеров инструкции «переименуй». Логика ветвления только в AUTOPILOT (единый источник).

**P1-6. `.business/INDEX.md` утверждает что `.business/` «не попадает в git».**

[.business/INDEX.md:4](../.business/INDEX.md#L4): «Папка `.business/` - скрытая, не попадает в git». Для исходного шаблона это ложь (всё tracked). После онбординга - правда (если P0-2 будет исправлен).

**Правка:** переформулировать: «папка `.business/` - приватная. После последнего шага онбординга Клод добавит её в `.gitignore` - твои данные не попадут в публичный репо».

[TEST-PROTOCOL.md:86](../TEST-PROTOCOL.md#L86). Файл будет удалён перед v1.0.0 (сам помечен), но до тех пор в git хранится. Security audit его не ловит, т.к. URL легитимный в allowlist. Низкая критичность.

### P3 - nice-to-have

**P3-1. AUTOPILOT шаг 3 не имеет варианта «Другой стек».**

Если пользователь на Elixir/Rust/Go - застрянет. Вариант «Ещё не знаю» не подходит по семантике.

**Правка:** добавить пункт «9. Другой - укажи какой».

**P3-2. AUTOPILOT шаг 5 vs пример в `claude-settings.json.example` - 4 против 9 правил.**

Согласовано через промпт 03-security.md («9 правил = 4 базовых + расширения»), но в AUTOPILOT формулировка «4 запрета безопасности» может смутить пользователя, увидевшего 9 строк в итоговом settings.json.

**Правка:** в AUTOPILOT шаге 5 добавить «4 концептуальных запрета, в `permissions.deny` это превратится в 9 regex-строк».

**P3-3. CLAUDE.md root не содержит ссылки на `prompts/INDEX.md` в таблице «Где что искать».**

[CLAUDE.md:49-59](../CLAUDE.md#L49-L59). В таблице есть пункты про `.business/`, `plans/`, но нет строки «готовые промпты под задачи → `prompts/INDEX.md`». В структуре репо упомянут, но в удобной навигационной таблице нет.

**Правка:** добавить строку.
