# План: вкладка «Система решений» (Горизонт 2)

## Цель
Дашборд показывает состояние. Эта вкладка показывает **что делать**, чтобы бизнес рос:
приоритизированный список действий с цифрой воздействия (₽/шт), конкретикой (имена, суммы)
и рекомендацией. Открыл - видишь, куда направить силы сегодня.

## Где
Новая вкладка «Система решений» в сайдбаре сразу под «Панель» (id `decisions`, иконка 🎯).
Функция `renderDecisions()`, вызов в `render()`.

## Источники данных (что уже есть в D)
- `D.orders.summary` - total_amount, total_payment, total_hired_cost, hired_profit
- `D.orders.doc_status` / `doc_by_decade` - документы по заказам (путёвка/проведён/реализация)
- `D.orders.by_customer` - неоплата = amount - payment (НЕ использовать balance - он раздут)
- `D.orders.lost_customers` - пропавшие во 2-й половине месяца
- `D.orders.by_hired_supplier` - маржа поставщиков найма
- `D.summary` - revenue/profit/margin своего парка, salesFakt/salesPlan
- `D.fleet` - ремонт/без водителя; `D.vehicles` - убыточные, штрафы

## Карточки решений (по убыванию денежного воздействия)
1. 🔴 **Собрать деньги** - ~34.6М не собрано; топ-6 должников с менеджером. Действие: обзвон.
2. 🔴 **Закрыть документы** - 523 заказа без полного пакета; фокус 136 своих без путёвки в декаде 21+. Без реализации нельзя выставить счёт.
3. 🟠 **Вернуть технику в работу** - 16 в ремонте + 5 без водителя; оценка потерь ~5.4М/мес.
4. 🟠 **Меньше найма, больше своих** - 11.6М найму (маржа ~23%) при 63% марже своих.
5. 🟠 **Вернуть ушедших клиентов** - 76 пропали; топ-6 по сумме с менеджером.
6. 🟡 **План продаж** - факт/план, прогноз к концу месяца, отстающие менеджеры.
7. 🟡 **Невыгодный найм** - поставщики с маржой <25% при заметном обороте.
8. 🟡 **Штрафы и убытки** - топ по штрафам, убыточные машины.

## Принципы
- Цифры брать надёжные: неоплата = amount-payment; документы - счётчики; не использовать раздутый balance.
- Каждая карточка: цвет по тяжести, число воздействия, конкретика, строка «→ Действие».
- Пустые/нулевые проблемы показывать зелёным «всё ок», не скрывать молча.
- Только чтение D, без запросов к серверу.

## Итог
Вкладка работает на боевых данных (июнь 2026). Баннер (4 плитки) + 8 карточек по приоритету.
Подтверждено через preview: 8 карточек, бейдж «2 горячих», реальные имена/суммы, ошибок нет.

**Находки в данных, учтённые в коде:**
- `total_balance` раздут (~15.6 млрд - повтор баланса орг. на каждой строке). НЕ использован.
  Дебиторка считается как `amount - payment` (надёжно).
- Документы: счётчики doc_status надёжны; декада 21+ показывает свежий всплеск (136 своих без путёвки).

отбрасывались. Урок: при сборке HTML конкатенацией проверять, что результат присоединяется.
