# Протокол тестирования (для Фазы 5)

**Этот файл — не для публикации.** До релиза v1.0.0 он удаляется из репо. Нужен для организованного прогона шаблона на разных нишах перед запуском.

## Метрика успеха

**4 из 5 тестеров доходят до конца AUTOPILOT за <120 минут без вмешательства ментора.** Если меньше 4 — итерируем AUTOPILOT и прогоняем снова.

## 5 ниш для тестирования

1. **Стоматология** — приложение записи пациентов (B2C услуги)
2. **SaaS для рекрутмента** — B2B софт с подпиской
3. **Агентство веб-разработки** — B2B услуги с проектной оплатой
4. **Внутренний инструмент отдела продаж** — не для продажи вовне (некоммерческий tool)
5. **Блог / медиа-проект без монетизации** — некоммерческий контент-проект

## Чеклист для каждого тестера

Перед тестом:

- [ ] VS Code установлен и открыт
- [ ] Claude Code extension установлен и авторизован
- [ ] Подписка Claude активна (Pro/Max)
- [ ] (РФ) VPN включён
- [ ] Есть чистая пустая папка для теста

Прогон:

- [ ] Склонировал репо через «Use this template» или `git clone`
- [ ] Открыл папку в VS Code
- [ ] Открыл Claude Code, написал любую фразу
- [ ] AUTOPILOT **запустился** с первой фразы (инвариант I1) ✅ / ❌
- [ ] Видно `.business/` в сайдбаре VS Code (инвариант I4) ✅ / ❌
- [ ] Прошёл шаги 1-7 (технические настройки) за ~30 минут
- [ ] Сделал паузу после шага 7 / или продолжил без паузы
- [ ] Если делал паузу: возобновил работу, Клод вспомнил шаг через `last_completed_step` (инвариант I3) ✅ / ❌
- [ ] Прошёл интервью (шаг 8) за ~60 минут
- [ ] После блока products получил Reality Check одним абзацем (инвариант I5) ✅ / ❌
- [ ] Интервью адаптировалось под выбранный тип проекта (инвариант I6) ✅ / ❌
- [ ] Сгенерировался `CLAUDE.md` под его проект (шаг 9)
- [ ] Прошёл тест-цикл (HELLO.md + план + ретро)
- [ ] Первый git-коммит создан (шаг 10)
- [ ] `AUTOPILOT.md` получил `completed: true`
- [ ] Пользователь понимает что в `prompts/` и как этим пользоваться

После прогона — опрос тестера:

1. Сколько минут заняло суммарно?
2. Где тебя запутало?
3. Что было непонятно в `.business/` плейсхолдерах?
4. Были ли вопросы, которые ты не мог адресовать ни Клоду, ни `TROUBLESHOOTING.md`?
5. Открывал ли `prompts/` после онбординга? Нашёл что искал?
6. Посоветовал бы шаблон другу? (1-10)
7. Что бы изменил?

## Собственный прогон Артемия (перед тестерами)

Прогони AUTOPILOT сам на чистом MacBook **до того** как даёшь тестерам. Ищи:

- Где Клод застревает на неоднозначной инструкции
- Где шаги идут не в том порядке
- Где промпты в `prompts/` ссылаются на файлы, которых ещё нет
- Где Reality Check слишком агрессивный или слишком мягкий
- Где интервью задаёт вопрос, на который уже был дан ответ в предыдущем блоке

Записывай проблемы в `retrospectives/2026-XX-XX_autopilot-selftest.md`. Потом итерируй AUTOPILOT.md и прогоняй снова.

## Где заканчивается тест

Тест пройден если тестер **самостоятельно** за <120 минут получил рабочее окружение и говорит: «я понял куда писать задачи и что делать дальше».

Если тестер в середине просит помощи у ментора — это провал: в реальности помощи не будет.
