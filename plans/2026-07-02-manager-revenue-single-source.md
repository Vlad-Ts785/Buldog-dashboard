# План: один источник выручки для менеджеров/зарплаты + помесячные планы

**Дата:** 2026-07-02
**Статус:** в работе

## Задача

Сейчас выручка менеджеров существует в двух источниках: "Менеджеры_данные" (отдельное
письмо от 1С, показывает только текущий месяц, без найма) и таблица заказов (`by_manager`,
уже содержит и свой парк, и найм, уже умеет архивировать по месяцам). Именно рассинхрон
между ними дал баг "Общая выручка показывает прошлый месяц" (2026-07-02).

**Решение (по просьбе Влада):** один источник факта - таблица заказов, везде. План
(цель по менеджеру) - величина, которую Влад вручную вводит КАЖДЫЙ МЕСЯЦ (не константа
в коде) - хранить в отдельном листе, по аналогии с Штатка_история/Данные_1С_история,
с учётом того, что план у истории (прошлый/позапрошлый месяц) может быть другим.

## Находки при подготовке

- Страница "Менеджеры" (`page-managers`/`renderManagers()`, читает `D.managers` из
  Менеджеры_данные) - **не подключена к сайдбару, мёртвый код**. Удаляем целиком, не
  переносим - не нужно.
- Реальные потребители "Менеджеры_данные": (1) `getSummaryData()` -> "Общая выручка" на
  Панели, (2) `buildManagersText()` - Telegram-сообщение по менеджерам/логистам.
- "По менеджерам"/"Зарплата" (`renderSalesMgr`, `calcDept`/`calcMgr`) уже читают
  `D.orders.by_manager` - план же берут из локальной JS-константы `MGR_PLANS`
  (не из Менеджеры_данные). Эту константу тоже убираем - план будет только в одном месте.
- `by_manager` уже содержит orders/amount/payment/profit/hired_*/own_amount - не хватает
  только "нал" отдельной суммой (тривиально добавить, поле `cash` уже есть в сырых данных).

## Хранилище планов

Новый лист **"Планы_менеджеров"**: Месяц (YYYY-MM) | Менеджер (фамилия) | План.
Влад редактирует вручную каждый месяц (просто дописывает новый блок строк на новый месяц,
старые не трогает - история сохраняется естественно, как и остальные листы-архивы).

## Риски

- Если Влад забудет добавить план на новый месяц - `.plan` будет 0 у всех, % выполнения
  станет бессмысленным (деление на 0 -> 0%). Не аварийный случай (не роняет дашборд),
  но заметный - решили не делать авто-фолбэк на предыдущий месяц, чтобы не маскировать
  забытый ввод (лучше явный 0%, чем тихо показать неверный план).
- getSummaryData() используется в нескольких местах (Панель, возможно Telegram-сводка
  buildSummaryText()) - проверено: buildSummaryText() читает Нормализованные_данные
  напрямую (парк, не менеджеры), не затронута.

## Осталось вручную (Владу)

1. Задеплоить `scripts/full_script_final.js`.
2. Один раз запустить `seedManagerPlansForCurrentMonth()` в редакторе Apps Script - создаст
   лист "Планы_менеджеров" и заполнит его текущими (июльскими) цифрами плана.
3. **Каждый новый месяц** - вручную дописывать в "Планы_менеджеров" новый блок строк
   (Месяц | Менеджер | План) на наступивший месяц. Без этого шага план будет 0% у всех.
