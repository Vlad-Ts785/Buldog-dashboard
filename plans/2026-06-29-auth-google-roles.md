# План: Авторизация через Google + персональные роли менеджеров

**Дата:** 2026-06-29
**Автор:** Клод (с Владом)
**Статус:** черновик

## Цель

Закрыть публичный доступ к дашборду (сейчас открыт всем по ссылке) и дать каждому
из 10 менеджеров персональную страницу с его данными вместо общего доступа ко всему.

## Контекст

Проверка показала: репозиторий публичный, GitHub Pages отдаёт `index.html` любому
без пароля, `doGet()` в Apps Script ([full_script_final.js:827](../scripts/full_script_final.js))
отдаёт весь JSON (выручка, прибыль, должники, заказы) без проверки токена. Это коммерческая
тайна, доступ нужен только Владу и его 10 менеджерам.

Решение: вход через Google (Google Identity Services) вместо своего логина/пароля -
у всех уже есть рабочие Google-аккаунты, не нужно хранить и проверять пароли самим.
Apps Script на сервере проверяет, кто пришёл, и решает, что ему показывать:
- Влад (админ) - видит всё, как сейчас.
- Менеджер - видит только свою страницу: его выручка, план, клиенты, показатель,
  рабочее окно (уточнить у Влада точный состав на фазе 3).

## Фазы

### Фаза 0. Стопгап - ключ доступа в API (готово, 2026-06-29)
- [x] `doGet()` требует `?key=...`, сверяется с `DASHBOARD_KEY` в Script Properties
- [x] Без ключа или с неверным - отдаёт `{"error":"Доступ запрещён"}`, не данные
- [x] Проверено напрямую curl-запросом к Apps Script URL - подтверждено
- Это временная защита одним общим ключом (для всех одинаковый). Фазы 1-5 ниже
  заменяют её на нормальную авторизацию с ролями по человеку.

### Фаза 1. Google Cloud OAuth Client ID (~30 мин, ручной шаг Влада)
- [ ] Влад создаёт OAuth Client ID в Google Cloud Console (тип "Web application"),
      добавляет origin `https://vlad-ts785.github.io`
- [ ] Client ID сохранить в `.env` (не в коде), прокинуть в `index.html` как конфиг

### Фаза 2. Лист доступа в Google Sheets (~20 мин)
- [ ] Новый лист «Доступ»: email | имя менеджера | роль (admin/manager)
- [ ] Заполнить 10 строк менеджеров + Влад как admin

### Фаза 3. Frontend: экран входа (~1.5 ч)
- [ ] Кнопка "Войти через Google" (Google Identity Services, без своих паролей)
- [ ] До входа - дашборд не рендерится, только экран логина
- [ ] ID-токен хранится в памяти/sessionStorage, прикрепляется к каждому запросу к API
- [ ] Обработка истёкшего токена - повторный показ экрана входа

### Фаза 4. Backend: проверка токена и фильтрация в Apps Script (~2 ч)
- [ ] `doGet(e)` принимает `id_token`, проверяет его через `https://oauth2.googleapis.com/tokeninfo`
      (подпись, audience = наш Client ID, срок действия)
- [ ] Без валидного токена или email не в листе «Доступ» - `doGet` отдаёт ошибку, не данные
- [ ] Email сверяется с листом «Доступ» -> определяется роль
- [ ] admin -> текущий полный `data` как сейчас
- [ ] manager -> новая функция `getManagerView(email)`: только его данные
      (выручка, план, клиенты, показатель, рабочее окно - состав уточнить с Владом)

### Фаза 5. Frontend: персональная страница менеджера (~3 ч)

Состав страницы (по списку Влада, 2026-06-29) - всё переиспользует логику,
уже посчитанную для «Системы решений» и «Продажи», просто отфильтрованную по имени менеджера:

1. **План и его выполнение** - факт/план текущего месяца, % выполнения, прогноз к концу
   месяца (как в `D.orders.summary`/`Продажи`, но только строки этого менеджера).
2. **Топ заказчиков, которые делают план** - топ клиентов текущего месяца по сумме
   (из `D.orders.by_customer`, фильтр по менеджеру).
3. **Топ заказчиков прошлого месяца** - на главном экране личной страницы, отдельным блоком.
4. **Кому позвонить** - клиенты, пропавшие во 2-й половине месяца (логика `lost_customers`,
   фильтр по менеджеру) - с рекомендацией "созвониться".
5. **Дебиторка по своим клиентам** - неоплата = amount-payment (НЕ balance, он раздут),
   список должников этого менеджера с суммами.
6. **Счётчики по заказам**: сколько строк-заказов в этом месяце, сколько закрыто
   (есть путевой лист + проведён + реализация), сколько ещё открыто.
7. **Воронка документов** - по каждому заказу: путевой лист (есть/нет) → проведён → реализация.
   Показывает где застряли документы, как в карточке "Закрыть документы" Системы решений,
   только по заказам этого менеджера.

- [ ] Для роли manager - скрыть весь сайдбар кроме его персональной вкладки
- [ ] Новая вкладка/вид с его данными (7 блоков выше)
- [ ] Для admin - всё как сейчас, без изменений в существующих вкладках, плюс
      возможность переключиться и посмотреть страницу любого менеджера (для контроля)

## Критерии готовности

- [ ] Открытие дашборда без входа - показывает только экран логина, без единой цифры
- [ ] `doGet()` без `id_token` или с чужим/просроченным токеном - отдаёт ошибку, не данные
- [ ] Тестовый менеджер видит только свою страницу, не видит чужие цифры и другие вкладки
- [ ] Влад как admin видит всё без изменений
- [ ] Нет ошибок в консоли / в Apps Script логах

## Риски

- Google tokeninfo endpoint - внешний запрос на каждый вызов `doGet`; если Google недоступен,
  дашборд не откроется. Смягчение: это краткосрочный сбой, не критично для внутреннего инструмента.
- Если у менеджера нет рабочего Google-аккаунта - придётся завести (разовая мелочь для 10 человек).
- Решение защищает данные на уровне API, но репозиторий GitHub остаётся публичным
  (сам HTML/JS код виден всем, не коммерческие цифры). Если это тоже критично -
  отдельное решение (приватный репо + GitHub Pro, либо перенос хостинга).
